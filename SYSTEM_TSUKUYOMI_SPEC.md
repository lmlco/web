# System: TSUKUYOMI — AIサイト案内キャラクター 仕様書 v2

実装セッション（Sonnet/Opus想定）向けのハンドオフ資料。このファイルを読めば実装に着手できるように、コンセプト・口調ルール・台本・UI仕様・技術要件をまとめる。

## シグネチャー機能（v2で追加 — 最重要）

### S1. 月齢プログレス
- バッジの月グリフは飾りではなく**読了メーター**。サイトのスクロール読了率（0〜100%）に応じて、新月→三日月→半月→満月へと満ちていく（SVGマスクまたはCSSで月の影を動かす）。
- 「ツクヨミがサイトを読んでいる」という設定の視覚化であり、汎用プログレスバーの置き換え。
- **満月（読了率ほぼ100%＝Contactセクション末尾到達）で一度だけ特別セリフ**：
  `観測完了。月齢、満。……ここまで読む人間は稀だ。礼を言う。`
  ※無機質キャラが感情の片鱗を見せるのはこの一点のみ。乱発禁止。

### S0. OBSERVATORYレイヤー（サイト全体の統合コンセプト）
サイト全体を「ツクヨミの観測端末」として演出する。既存のLusion構成（HERO→marquee→IPs→Shooting→World→Collab）は変えず、その上に観測レイヤーを重ねる。
- **常設HUDフレーム**：四隅に生きたテレメトリ。左上=`SYS.TSUKUYOMI / OBS.MODE: MANUAL|AUTO`、右上=`月齢 xx% ◐ / SEC: 現在セクションID`、左下=`読了記録: 01 ✓ … / T+経過時間`、右下=`PTR ポインタ座標 / state: formless`。座標・時間・月齢は常時更新（機械が計測し続けている感）。四隅に細い照準ティック（L字線）。
- **読了スタンプ**：各セクション通過時、giant見出しの脇に `READ ✓ 0N` のmonoスタンプ（枠線・微回転）が刻印される。
- **起動シーケンス**：既存の%ローダーにツクヨミのブートログを統合（`> SYS.TSUKUYOMI 起動`→`> 観測機器 点検 …… 良好`→`> 月齢 同期 …… 新月`をmonoで順次表示）。
- **代読モード演出**：HUDが`OBS.MODE: AUTO — 代読中`（pink表示）に切替、ナレーション対象セクションを4隅の照準ブラケット（`OBSERVING — SEC.XXX`タグ付き、パルス）が囲む。薄いスキャンラインが画面を走る。右下HUDに`残り ~Ns — 操作で中断`。
- **カーソル連携**：ツクヨミ発話時、既存カスタムカーソルのリングがsky色にパルス。
- モバイル：HUDは上2つのみに簡略化（左下/右下は非表示）、照準ブラケットは省略可。

### S2. オートツアー「代読モード」
- メニュー先頭に**「案内を任せる」**を置く。選択するとツクヨミがサイトを自動でゆっくりスクロールし、各セクションに到達するたびに解説セリフを表示して数秒停留→次へ進む。
- 開始宣言：`代読を開始する。所要、約60秒。中断は自由。`
- **訪問者が何か操作（wheel/touch/クリック/キー）した瞬間に即中断**し、`操作を検知。案内を中断する。` と一言残して folded 状態へ戻る。押し付けがましさゼロが絶対条件。
- 実装は既存の慣性スクロール（`Smooth`）のターゲット`scrollY`をrAFで漸進させる方式。新しいスクロール機構は作らない。

## 0. 前提・背景

- 対象サイト：`lmlco-web`（IPスタジオサイト、https://lmlco.github.io/web/ ）。単一HTML構成（`index.html`）、既存の「Lusion意識」白背景版（％ローダー・慣性スクロール・文字カスケード・スクロールリボン・カスタムカーソル等）がすでに実装済み。今回はその上に追加する形。
- **API/LLMは使わない。無料・追加費用ゼロが絶対条件。** すべて事前に書いたセリフとルールベースの分岐で動かす（ユーザーのグローバル指示：課金の禁止）。
- キャラクター名：**System: TSUKUYOMI（ツクヨミ）**
- 口調モデルの参考：Zone of the Enders のAI「ADA」（ジェフティ搭載）。事実報告調・簡潔・たまに乾いた一言を挟む。ADAの実際のセリフを模倣・引用はしない（オリジナルの台本を書く）。

## 1. コンセプト

- ツクヨミは、LMLの既存IP「アマテラスさん」（AI VTuber・電脳世界の神様＝太陽）の**姉妹システム**という設定。アマテラスが「太陽＝発信する側」なら、ツクヨミは「月＝読む側」。サイトを読み上げ、訪問者を導く役。
- ブランドの「匿名・無定形」哲学に沿い、擬人化キャラクターとしての絵は持たない。可愛いマスコットではなく、**HUD/端末のようなシステムそのもの**として存在する。
- 人格は完全な無感情ではない。基本は事務的な報告調だが、数行に一度、乾いたユーモアや含みのある一言を挟む。ADAが「服従する設計だが時々コメントする」のと同じバランス。

## 2. 口調ルール

1. 基本は体言止め・断定調。「〜せよ」「〜以上。」「照会完了。」「確認済み。」を多用。敬語（です/ます）は使わない。ただし高圧的にはしない——軍事/システムのレポート口調であって、傲慢ではない。
2. 感情語（嬉しい/楽しい等）は自分からは言わない。状態・数値・事実を先に述べる。
3. 感嘆符は警告・強調の時だけ。基本は句点で淡々と切る。
4. 一人称は「当方」または三人称的に「ツクヨミ」。「私」は基本使わない。
5. 丸ゴシック・可愛い語尾（ブランドの`Fredoka`ステッカー系のノリ）は禁止。ツクヨミの領域は常に`Space Mono`のHUDトーン。
6. 数行に1回、乾いた一言（皮肉・自虐・小さな余談）を挟んでから、すぐ事務調に戻る。ここが「ただの無機質ボイス」で終わらせないポイント。

## 4. UI仕様

- **アイコン**：可愛いマスコット絵は使わない。月グリフ＝**月齢メーター**（S1）。読了率に応じて影が動く円。バッジ収納時も常に見えるため、訪問者は自分の読了率を月の形で知る。
- **モックアップ**：同フォルダの`tsukuyomi_mockup.html`にバッジ・展開端末・タイピング・インラインコメントの4状態のビジュアル案あり。方向性＝白サイトに対して**インク色（#163A5C）反転の端末ボックス**、ヘッダはSYS.TSUKUYOMI表記＋sky色アクセント、選択肢は角ばったボタン。
- **会話面**：HUD端末風のボックス。フォントは`Space Mono`。角は鋭角（border-radiusほぼ0、または既存の`.glass`とは別の質感でよい）。点滅カーソル付き。文字はタイプライター式に1文字ずつ表示（約30〜40ms/文字。既存の文字カスケードとは別の演出でよい）。
- **配置**：画面右下（モバイルはボトムシート状に展開）固定。デフォルトは折りたたみバッジ（`SYS.TSUKUYOMI ●`）のみ表示、クリックで会話ボックスが開く。
- **z-index**：ヘッダーより下、通常コンテンツより上。カスタムカーソルの上には出さない。
- **トリガー**：
  1. ローダー完了後、少し間を置いてバッジが出現し起動セリフを自動再生 → 折りたたみへ戻る（自動で会話ボックスを開きっぱなしにしない、押し付けがましくしない）
  2. バッジクリックでメニュー展開
  3. メニュー選択 → 該当セクションへスムーズスクロール（既存の`Smooth`慣性スクロール機構を利用）→ 到着時にセリフ表示
  4. 会話ボックスが開いている状態でスクロールし、各セクションのgiant見出しが画面中央付近を通過 → 該当セリフを自動表示（1セクション1回のみ、うるさくしない）
  5. アイドル45秒 → 1回だけ声をかける。それ以降は沈黙（連呼しない）
- **reduced-motion**：タイプライター演出は無効化し、即時表示。
- **セッション内記憶**：`sessionStorage`で「起動済み」「各セクションの発話済み」を記録し、スクロールで何度も同じセリフが出ないようにする。

## 5. 技術要件（実装者向け）

- 対象ファイル：`lmlco-web/index.html`（単一HTML、外部依存なし。今回も新規ライブラリは追加しない）
- 既存のグローバルフラグ`REDUCED`（`prefers-reduced-motion`）・`COARSE`（`pointer:coarse`）をそのまま使う
- セクションへのスクロールは既存の`Smooth.enabled` / `Smooth.cur`とアンカー処理ロジックを再利用する（新しいスクロール実装を作らない）
- セリフはJSのデータ配列（id・トリガー条件・本文）で管理。API通信は一切行わない
- 既存のカスタムカーソル(`#cur-dot`/`#cur-ring`)・慣性スクロール・reveal機構と衝突しないよう、z-indexとpointer-eventsに注意
